# Landing Page Psychology Audit Rubric

Use this rubric to turn the canonical reference into specific landing page decisions.

## Principle Checks

| Principle | Inspect | Improve |
| --- | --- | --- |
| Serial position effect | The first screen and final CTA section | Put the strongest claim in the hero and the strongest proof, urgency, or reassurance before the final CTA. Let the middle carry details. |
| Von Restorff effect | CTA contrast, button hierarchy, repeated visual patterns | Make the primary CTA visibly distinct from the page's surrounding color, shape, or rhythm while staying brand-consistent. |
| Cognitive load theory | Section density, bullets, feature grids, comparison blocks | Keep each section to 3-4 main points. Move overflow into supporting detail, icons, tabs, or progressive disclosure. |
| Social proof cascading | Order, variety, and spacing of proof elements | Layer proof naturally: immediate logos or scale cue, testimonial, feature proof, case study or metric, final CTA reassurance. |
| Anchoring bias | First number, scale cue, problem size, pricing context | Lead with the largest credible anchor: customer scale, market size, revenue at risk, time wasted, or category-level problem. |
| Completion bias | Scroll flow, story continuity, forms, progress cues | Make the page feel like a path with a beginning, middle, and end. Use reveals, progress, or "next section" cues only when they help continuation. |
| Loss aversion framing | Hero promise, problem statement, feature benefits | Frame value around what the visitor is currently losing when that is more concrete than the future upside. |
| Halo effect | Typography, imagery, motion, spacing, UI polish | Treat visual quality as trust evidence. Improve the highest-visibility visual details first: hero, CTA, proof cards, screenshots, and typography. |
| Choice reduction | CTA count, nav links, pricing paths, secondary actions | Keep one dominant action. If a second path is necessary, make it visually secondary and clearly different in purpose. |

## Audit Output Shape

Use this format for page reviews:

| Priority | Finding | Principle | Exact fix |
| --- | --- | --- | --- |
| High/Medium/Low | What is weakening the page | Which principle explains it | The concrete copy, layout, proof, or visual change |

Then add a short `Recommended Page Flow` if the section order should change.

## Page Plan Shape

Use this format for new pages:

1. Hero: strongest claim, primary CTA, first anchor, immediate proof
2. Proof strip: logos, user count, category proof, or trust marker
3. Problem/value section: current loss and why it matters now
4. Feature/detail sections: 3-4 points per section
5. Testimonial or case study: specific proof with a human quote or metric
6. Objection handling: risk reversal, FAQ, security, process, implementation, pricing clarity
7. Final CTA: strongest remembered proof plus one primary action
